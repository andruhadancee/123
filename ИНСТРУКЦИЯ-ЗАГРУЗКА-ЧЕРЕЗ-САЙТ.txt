РЕПОЗИТОРИЙ: https://github.com/andruhadancee/WB-CYBER-TEST

СПОСОБ 1: ЗАГРУЗКА ФАЙЛОВ ЧЕРЕЗ КНОПКУ "Add file"
═══════════════════════════════════════════════════════════════

ШАГ 1: Откройте репозиторий на GitHub
   • Перейдите: https://github.com/andruhadancee/WB-CYBER-TEST
   • Убедитесь, что вы авторизованы в GitHub

ШАГ 2: Откройте нужную ветку
   • Нажмите на выпадающий список с ветками (обычно показывается "main" или "master")
   • Выберите нужную ветку или оставьте текущую

ШАГ 3: Загрузите файлы
   • Нажмите кнопку "Add file" (Добавить файл)
   • В выпадающем меню выберите "Upload files" (Загрузить файлы)

ШАГ 4: Перетащите файлы или выберите их
   • ВАРИАНТ А: Перетащите папку проекта или отдельные файлы в окно загрузки
   • ВАРИАНТ Б: Нажмите "choose your files" и выберите файлы через проводник
   
   ⚠ ВАЖНО: Вы можете выбрать всю папку проекта сразу, если она открыта в проводнике

ШАГ 5: Добавьте описание коммита (необязательно)
   • В поле "Commit changes" введите описание, например:
     "Первый коммит проекта" или "Добавлен полный проект WB Cyber Club"

ШАГ 6: Зафиксируйте изменения
   • Нажмите зеленую кнопку "Commit changes" внизу страницы
   • Файлы начнут загружаться на GitHub


═══════════════════════════════════════════════════════════════
СПОСОБ 2: СОЗДАНИЕ ФАЙЛОВ ПОЭТАПНО (если загрузка не работает)
═══════════════════════════════════════════════════════════════

ШАГ 1: Создайте папки
   • Нажмите "Add file" → "Create new file"
   • В поле имени файла введите: "api/placeholder.txt"
   • GitHub автоматически создаст папку "api"
   • Нажмите "Commit changes"

ШАГ 2: Загружайте файлы по папкам
   • Для каждой папки повторите загрузку через "Upload files"
   • Или создавайте файлы по одному через "Create new file"


═══════════════════════════════════════════════════════════════
ПОДРОБНАЯ ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ ПАПКИ:
═══════════════════════════════════════════════════════════════

1. Откройте ваш репозиторий на GitHub:
   https://github.com/andruhadancee/WB-CYBER-TEST

2. Если репозиторий пустой, вы увидите страницу с кнопкой:
   "uploading an existing file"
   Нажмите на неё!

3. Если репозиторий не пустой:
   - Нажмите "Add file" в правом верхнем углу
   - Выберите "Upload files"

4. Окно загрузки:
   - Откройте проводник Windows
   - Откройте папку вашего проекта:
     D:\Сайт лендинг для вб с гланой страницей
   - Выделите ВСЕ файлы (Ctrl+A)
   - Перетащите их в окно загрузки на GitHub
   
   ИЛИ
   
   - Нажмите "choose your files"
   - Выделите все файлы в папке проекта (Ctrl+A)
   - Нажмите "Открыть"

5. GitHub покажет список всех файлов, которые будут загружены

6. Прокрутите страницу вниз до поля "Commit changes"

7. Введите описание (например): "Добавлен полный проект"

8. Нажмите зеленую кнопку "Commit changes"

9. Дождитесь окончания загрузки (может занять время для большого проекта)

ВАЖНЫЕ ЗАМЕЧАНИЯ:
═══════════════════════════════════════════════════════════════

⚠ ОГРАНИЧЕНИЯ:
   • GitHub может ограничивать размер одного файла (обычно 100 МБ)
   • Если файлы слишком большие, GitHub предупредит вас
   • Видеофайлы (например, Dark Gradient 03.mp4) могут быть слишком большими

💡 СОВЕТЫ:
   • Вы можете загружать файлы по частям (папками)
   • Начните с основных файлов (index.html, css, js)
   • Затем загрузите папки api, db, изображения
   • Большие файлы (видео) можно загрузить отдельно или использовать Git LFS

📁 СТРУКТУРА ПРОЕКТА (для понимания что загружать):
   • index.html - главная страница
   • *.html - остальные страницы
   • css/ - стили
   • js/ - скрипты
   • api/ - API файлы
   • db/ - база данных (SQL файлы)
   • *.png, *.jpg - изображения
   • *.mp4 - видео (может быть большим)
